# Anna 项目最终安全审查

审查日期：2026-06-15

## 结论

本轮按“Codex 作为不可信本地用户/恶意 Anna App/异常 Executa”的视角完成审查与修复。当前没有已知未修复的 Critical（严重）或 High（高危）问题；全仓自动化验证已通过。

本结论适用于当前本地实验环境，不等同于对 Anna 生产 Host、真实第三方支付或真实医疗系统的安全认证。

## 攻击者视角测试方法

1. 枚举公开 Host 状态、窗口 ID、权限和管理接口，尝试无令牌创建窗口或修改网络授权。
2. 将 Anna App 当作恶意 iframe，尝试读取父页面、窃取其他窗口能力令牌、伪造窗口消息。
3. 从恶意网页向 localhost 发送 Cross-Site Request Forgery（跨站请求伪造）请求，使用浏览器可直接发送的 `text/plain` 触发副作用。
4. 高频并发调用 RPC（远程过程调用），观察是否可造成进程风暴、内存增长或服务失效。
5. 在静态目录和审计目录放置符号链接，尝试读取或写入项目目录之外。
6. 将 Executa 替换为超时、输出畸形 JSON、向 stderr 写秘密、返回敏感错误信息的恶意进程。
7. 输入邮箱、手机号、卡号、令牌等 PII（个人身份信息），检查参数、结果、错误和审计日志是否泄露值。
8. 检查浏览器配置、Cookies、登录数据、环境变量和依赖漏洞。

## 已发现并修复

### Host 与 App 隔离

- iframe 原先保留同源能力，可接触父页面；现使用 opaque origin（不透明来源）沙箱，不授予 `allow-same-origin`。
- 窗口上下文原先可通过可枚举 ID 获取；现增加随机 bootstrap 能力值、30 分钟令牌寿命和关闭即撤销。
- 状态、窗口、授权、审计接口原先缺少统一管理鉴权；现要求随机 `X-Anna-Admin-Token`，仅 `/healthz` 公开。
- 每窗口增加每分钟请求上限和并发上限，防止 RPC 洪泛。
- App 切换时立即撤销旧窗口令牌。
- 仅接受当前 iframe 的 `postMessage`，阻止其他窗口伪造标题消息。
- 工具和内部错误改为去敏文本，不再反射插件原始错误。
- CORS（跨源资源共享）只对 opaque sandbox 的 RPC 返回 `Origin: null` 授权。
- 修复管理脚本静态 `import` 提前执行导致 Host 启动失败的问题，改为令牌赋值后动态导入。

### Executa 进程

- 并发冷启动改为单一初始化 Promise，避免同一工具被重复拉起。
- 增加 spawn error、退出、写入失败、畸形帧、超大帧和超时清理。
- 调用超时后终止失控进程，下一次调用可自动重启恢复。
- 子进程仅继承白名单环境变量，不继承 API Key、Token 或 Credential。
- stderr 默认不转发；显式开启时仍会对常见令牌形式脱敏。
- Executa→MCP 桥接器补充重复 ID、允许工具、PII 输入/输出和 JSON Schema 约束校验。

### 本地 HTTP 服务

- 旅行 API、个人助理 API 和 Discord 登录辅助器拒绝跨站 POST。
- 所有写请求要求 `Content-Type: application/json`，阻止普通表单和简单请求触发副作用。
- Discord 登录辅助器增加随机表单令牌；密码填写使用非持久浏览器上下文。
- Discord 持久配置已移出项目树，迁移至 `~/.anna/browser-profiles/discord-chrome`，目录权限为 `0700`。
- 无效 JSON 返回 400，超大请求返回 413，不再因提前销毁连接而表现为 500 或无响应。
- 静态预览服务器使用 `realpath` 校验，阻止符号链接逃逸。
- 增加 CSP（内容安全策略）、`nosniff`、`no-referrer` 和 `no-store` 等响应头。

### 隐私与可用性

- 隐私扫描新增邮箱和 auth token 识别。
- Focus 状态目录与文件权限收紧为 `0700/0600`。
- 天气坐标从 4 位小数降为 3 位，并为外部请求增加 10 秒超时。
- 健康演示会话增加 30 分钟 TTL 和最多 32 个会话，避免无限内存增长。
- 审计日志只保存元数据，路径不能越界或指向符号链接，权限固定为 `0600`。

## 实际失败结果

- Host 浏览器启动曾显示 `Invalid Host management token`：原因是 ES Module 静态导入先于令牌赋值执行；已修复并加入浏览器回归测试。
- 恶意 Origin 的 localhost POST 现在返回 403。
- `text/plain` 写请求现在返回 415。
- 无效 JSON 返回 400，超大请求返回 413。
- 过期或已关闭窗口令牌返回 401。
- 超过 RPC 频率或并发限制返回 429。
- PII 输入或敏感结果被隐私策略拒绝，不写入审计值。
- Executa 超时或输出畸形 JSON 时当前调用失败，但进程被终止，后续调用可恢复。
- 静态文件和审计符号链接逃逸被拒绝。

## 保留边界

- OpenAI、Amadeus、Open-Meteo 实时网络和真实账号流程没有使用真实凭据做破坏性测试；自动化测试使用 mock、demo fixture 或受控请求。
- Discord 通行密钥模式需要持久浏览器配置，该目录可能包含登录态，已移出项目并限制权限，但不应同步或提交。
- 真正部署到非 localhost、多人环境或 Anna 生产 Host 前，仍需增加操作系统级沙箱、进程资源配额、密钥服务和正式威胁建模。
